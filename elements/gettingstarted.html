Getting Started with Git

Introduction

This guide will walk you through the most essential Git commands and how to get started with Git, once installed on your local machine.

1. Setting up Git

Before you start using Git, you need to configure your identity so that your commits are tied to your name and email. This only needs to be done once per machine.

Step 1: Configure Your Git Identity

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Replace "Your Name" with your actual name and "your.email@example.com" with your email address.

Now Git knows who you are, and every commit you make will be linked to your identity!

2. Starting a Repository

a. Initialize a Git Repository

Once your identity is set up, you can start tracking changes in your project. To do this, navigate to your project folder in your terminal or command prompt and initialize Git:

git init

This creates a .git folder inside your project where Git stores the history of your changes.

b. Cloning an Existing Repository

If you want to copy an existing project from a remote repository, you can use the git clone command:

git clone <repo-url>

This will download all the project files along with the entire history of the project from the remote repository to your local machine.

3. Tracking Changes

a. Check the Status

To see which files have changed, which files are staged for commit, and which files are untracked, use:

git status

b. Stage Files

Before committing changes, you need to add the changes to the staging area. This tells Git which files you want to include in the next commit. To stage specific files:

git add <filename>

To stage all changes at once, use:

git add .

c. Commit Changes

Once you’ve staged the files, create a commit to record your changes with a descriptive message:

git commit -m "Descriptive message of the changes"

This creates a snapshot of your project at this point in time, allowing you to track your progress.

d. Push Changes

Once you’ve commited the files, you can push them to your remote repository

git push

This piushes the code to the remote repository

4. Viewing Commit History

To see the history of commits in your project, use the following command:

git log

This will display a list of all previous commits, showing their commit IDs, author names, and commit messages.
