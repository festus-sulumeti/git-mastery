Comprehensive Git Commands Guide

1. git init

Initialize a new Git repository in your project directory.

git init

This command creates a new local Git repository. You can use it to turn an existing project into a Git repository.

2. git clone

Clone an existing repository from a remote server.

git clone [url]

This command copies a remote repository to your local machine, including all of the files, commits, and branches.

3. git status

Check the status of your working directory and staging area.

git status

This command shows the current state of the working directory, indicating files that are staged, modified, or untracked.

4. git add

Add changes to the staging area.

git add [file]

This command stages changes (file modifications, additions, or deletions) for the next commit. Use git add . to stage all files.

5. git commit

Commit staged changes with a descriptive message.

git commit -m "commit message"

This command saves a snapshot of your project history. Always include a descriptive commit message to explain the changes.

6. git push

Push committed changes to a remote repository.

git push [remote] [branch]

This command uploads your local repository content to a remote repository, allowing others to access the changes.

7. git pull

Fetch and integrate changes from a remote repository.

git pull [remote]

This command retrieves updates from the remote repository and merges them into your local branch.

8. git branch

Create, list, or delete branches in your repository.

git branch

This command helps you manage branches. To create a new branch, use git branch [branch-name]. To delete, use git branch -d [branch-name].

9. git checkout

Switch to a different branch or commit.

git checkout [branch-name]

This command allows you to switch between different branches or even revert to a specific commit.

10. git merge

Merge changes from one branch into another.

git merge [branch]

This command integrates changes from another branch into the current branch.

11. git log

View the history of commits in the repository.

git log

This command displays a list of commits, showing the commit hash, author, date, and commit message.

12. git reset

Undo changes in the working directory or staging area.

git reset [file]

This command removes files from the staging area, effectively undoing the last git add.

13. git stash

Temporarily save changes that you don’t want to commit right away.

git stash

This command saves your changes in a “stash” and restores a clean working directory. You can apply the changes later using git stash apply.

14. git remote

Manage your remote repository connections.

git remote -v

This command displays the remote connections. To add a new remote, use git remote add [name] [url].

15. git diff

Show changes between commits, branches, or your working directory.

git diff

This command shows differences between the working directory and the staging area. It is useful for reviewing changes before committing.
